## Características clave a documentar

1. **Atomic Design Pattern**
   - Qué es cada nivel (Atom, Molecule, Organism, Template, Page)
   - Cuándo usar cada uno
   - Ejemplos de cada nivel

2. **Estructura de carpetas**
   ```
   src/
     components/
       atoms/
         ComponentName/
           ComponentName.tsx
           ComponentName.module.css   # ← NUEVO
           ComponentName.test.tsx
           index.ts
   ```

3. **CSS Modules Pattern** (NUEVO)
   ```tsx
   // Importar estilos
   import styles from './ComponentName.module.css';

   // Helper function
   const getClassName = (...classes: (string | undefined | false)[]) =>
     classes.filter(Boolean).join(' ');

   // Usar en componente
   <div className={getClassName(styles.container, isActive && styles.active)}>
   ```

4. **Naming conventions**
   - PascalCase para componentes
   - camelCase para funciones, variables y clases CSS
   - kebab-case para archivos CSS (excepto .module.css)
   - Sufijos específicos (.tsx, .test.tsx, .module.css)

5. **Tokens del design system**
   - Colores: #DA291C (rojo Claro), #006081 (azul), etc.
   - Tipografía: 'Open Sans'
   - Espaciado: 4px, 8px, 12px, 16px, 20px, 24px, 32px
   - Border radius: 2px, 5px, 10px
   - Shadows

6. **Testing patterns**
   - Usar @testing-library/react
   - Tests de renderizado
   - Tests de interacción
   - Tests de accesibilidad
   - **NO verificar clases CSS** (fallan con mocks)

7. **Import/Export patterns**
   - Siempre exportar desde index.ts
   - Named exports vs default exports
   - Re-exports organizados
   - CSS Module siempre último en imports
